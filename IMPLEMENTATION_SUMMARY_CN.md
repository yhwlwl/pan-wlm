## 🔐 权限系统集成

### 权限检查链路

```
用户点击"批量下载"
    ↓
前端检查：canDownload 权限?
    ├─ 无 → ❌ 提示"无下载权限"
    └─ 有 → 继续
        ↓
    分离文件和文件夹
        ├─ 文件 → 原有下载逻辑
        └─ 文件夹 → 调用 ZIP API
            ↓
        后端权限检查（对每个路径）
            ├─ 验证用户身份
            ├─ 检查 view 权限
            ├─ 检查 download 权限
            ├─ 检查 basePath 隔离
            └─ 任一失败 → ❌ 403 Forbidden
            ↓
        ✅ 生成 ZIP 并下载
```

### 权限场景矩阵

| 角色 | view | download | 能否下载文件 | 能否下载文件夹 |
|------|------|----------|----------|------------|
| admin | ✅ | ✅ | ✅ | ✅ |
| manager | ✅ | ✅ | ✅ | ✅ |
| guest (无权) | ✅ | ❌ | ❌ | ❌ |
| 受限 basePath | ✅* | ✅* | ✅* | ✅* |

*仅限于 basePath 内的路径

## 🚨 故障排查

### 问题 1：archiver 不可用
```
错误提示：503 Service Unavailable
解决：npm install archiver
```

### 问题 2：文件夹下载返回 403
```
错误提示：权限不足
原因：缺少 download 权限或 basePath 限制
解决：检查用户权限配置
```

### 问题 3：ZIP 文件为空或很小
```
原因：部分文件权限限制，自动跳过
解决：检查单个文件的权限
```
