# 批量下载没反应 - 调试指南

## 🔧 快速排查步骤

### 第 1 步：打开开发者工具并查看日志

1. **打开浏览器**
2. **按 F12 或右键 → 检查** 打开开发者工具
3. **切换到 Console 标签页**
4. **刷新页面**

### 第 2 步：尝试选择文件

1. 在网盘界面，点击任何一个文件或文件夹前的复选框 ☐
2. **查看 Console 中是否出现日志**：
   ```
   [复选框] 选中: 文件名.txt
   [复选框] 当前选中项总数: 1
   ```

**预期结果**：应该看到选中的日志
**如果没有日志**：表示复选框事件没有被触发

---

### 第 3 步：点击批量下载按钮

1. 选中至少 1 个项目后，底部状态栏应显示"↓ 批量下载"按钮
2. **点击批量下载按钮**
3. **查看 Console 中的下列日志**：

```
[批量下载] 触发，选中项: Set(1) { '文件名.txt' }
[批量下载] 权限检查: { canDownload: true, ... }
[批量下载] 云盘类型: { prov: 'baidu', isBaidu: true, ... }
[批量下载] 项目分类: { 文件数: 1, 文件夹数: 0 }
[批量下载] 下载文件: 文件名.txt
```

---

## 🐛 常见问题和解决方案

### 问题 1：复选框点击后 Console 没有日志

**症状**：
```
[复选框] 选中 日志没有出现
```

**原因**：复选框 onclick 事件没有被正确绑定

**解决**：
1. **重新构建项目**：
   ```bash
   npm run build
   npm run dev  # 本地测试
   # 或
   npm start    # 生产环境
   ```
2. **清除浏览器缓存**：Ctrl+Shift+Delete（后选择"缓存的文件"）
3. **硬刷新页面**：Ctrl+Shift+R（Windows）或 Cmd+Shift+R（Mac）

---

### 问题 2：选中项显示为 0，但 Console 显示选中了

**症状**：
```
[复选框] 选中: 文件.txt
[复选框] 当前选中项总数: 1
但页面上状态栏还显示"5 个项目"，没有显示"1 个选中项"
```

**原因**：React 状态更新延迟或 re-render 问题

**解决**：
1. **等待 1-2 秒**，让 React 完成状态更新
2. **点击其他文件**，再点击回来测试是否正常
3. **切换目录后返回**

---

### 问题 3：无权限错误

**症状**：
```
[批量下载] 触发，选中项: Set(1) {...}
[批量下载] 权限检查: { canDownload: false, ... }
提示消息：❌ 无下载权限
```

**原因**：用户没有下载权限

**解决**：
1. **确认你已登录**（页面左上角应显示用户名）
2. **使用管理员账号**检查当前用户的权限
3. **查看管理后台** → 用户权限，确保下载权限开启

---

### 问题 4：选中文件夹但没有 ZIP 下载

**症状**：
```
[批量下载] 项目分类: { 文件数: 0, 文件夹数: 1 }
[ZIP下载] 打包路径: ['/文件夹名']
但没有后续的下载反应
```

**可能原因**：
1. 后端 `/api/alist-zip-download` API 没有响应
2. archiver 库未安装

**解决**：
1. **检查后端日志**（服务器控制台）是否有错误
2. **验证 archiver 安装**：
   ```bash
   npm list archiver
   # 应该输出：
   # └── archiver@6.x.x
   
   # 如果未安装，运行：
   npm install archiver
   ```
3. **重启应用**：
   ```bash
   npm start  # 或 npm run dev
   ```

---

### 问题 5：混合选择（文件+文件夹）不工作

**症状**：
```
[批量下载] 项目分类: { 文件数: 2, 文件夹数: 1 }
[批量下载] 下载文件: 文件1.txt
[批量下载] 下载文件: 文件2.pdf
[ZIP下载] 打包文件夹: ['文件夹1']
但没有看到任何下载
```

**可能原因**：
1. 文件直接下载但没有提示（浏览器下载默认）
2. ZIP 下载因版本问题失败

**检查方式**：
1. **查看浏览器下载历史**（Ctrl+J）是否有文件
2. **查看 Network 标签页**（F12 → Network）是否看到请求
   - 搜索 "alist-zip-download"
   - 检查响应状态码（200 = 成功，403 = 权限问题，503 = 库未安装）

## 🔍 详细的网络请求检查

### 如何检查 ZIP 下载 API 是否返回正确结果

1. **打开 F12 开发者工具**
2. **切换到 Network 标签**
3. **勾选"Preserve log"选项**
4. **点击批量下载**
5. **在 Network 列表中搜索**"alist-zip-download"
6. **点击该请求**
7. **查看下列几个标签**：

#### "Headers" 标签 - 检查请求是否正确
```
Request URL: /api/alist-zip-download?paths=[...
Request Method: GET
Status: 200 ✅（应该是 200）

重要 Headers:
- Authorization: Bearer token... (应该有身份令牌)
```

#### "Preview" 标签 - 检查响应内容
```
✅ 正常：二进制数据（看不到可读内容）
❌ 错误：看到错误信息文本，如：
   - "路径 /xxx 禁止访问或下载"
   - "ZIP 压缩功能暂不可用"
   - "请先登录"
```

#### "Response" 标签 - 检查原始响应
```
✅ 正常：看到 \x50\x4B (ZIP 文件的二进制标识)
❌ 错误消息：查看具体错误提示
```
